# Changelog

## [1.0.0] - 2026-06-23

### Highlights

- Rebranded the client as BOOST for user-facing application names, VPN session labels, Android service notifications, package metadata, release files, and documentation.
- Connected the account, dashboard, subscription, package, order, and device flows to `https://new.moneyfly.top/api/v1`.
- Added account login, registration, password reset, package purchase, order status, subscription status, and device management against the MoneyFly backend.
- Automatically imports the logged-in user's subscription into the local profile list.
- Uses the backend sing-box subscription URL (`type=singbox`) so BOOST receives JSON configuration instead of the generic plain-text subscription.
- Replaced manual profile addition with account-based subscription sync.
- Uses the `boost://` deep-link scheme for BOOST import links.

### Fixed

- Fixed account subscriptions that opened the generic subscription URL and caused `unable to determine config format`.
- Fixed login refresh handling so expired access tokens can be refreshed before account data and subscription sync run.
- Fixed duplicate account subscription profiles by replacing previous account-owned profiles with the active backend subscription.
- Fixed GitHub release publishing so GitHub Actions creates releases with English release notes and uploads generated assets.

### Build And Release

- Versioning starts at `1.0.0`.
- Release artifacts are generated by GitHub Actions instead of being stored in the repository.
- GitHub Releases include English change summaries, Android, macOS, Windows, Windows 7 legacy packages, checksums, and appcast metadata.
